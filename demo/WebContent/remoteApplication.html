Конфигурирование удаленного приложения

Для обеспечения на удаленном сервере вызова методов его функций в его конфигурацию необходимо добавить секцию <connector> и в этой секции опубликовать одну или несколько локальных функций при помощи секций <publish>.

Примеры секций <connector> для различных протоколов с публикацией в них одной локальной функции <info-function>, которая будет известна другим системам под именем <info>. В атрибутах секций <connector> определяются параметры коннектора, в частности для протокола RMI порт и имя по которым удаленные системы могут соединится с эти коннектором. Для протокола HTTP достаточно определить только имя коннектора. 

	<connector class="RMIConnector" name="rmi-connector" port="23000" >

		<publish name="info" function="info-function" />

	</connector>

	<connector class="HTTPConnector" name="http-connector" >

		<publish name="info" function="info-function" />

	</connector>

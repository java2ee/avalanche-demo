Реализация адаптера

Для обеспечения вызовов методов функции необходимо определить адаптер. Определение адаптера сводится к декларации интерфейса с методами функции. Декларация методов адаптера обязательно должна содержать исключение AvalancheRemote. Нет необходимости декларировать все методы функции. Достаточно определить только те методы, которые планируется вызывать с использованием этого адаптера.

Пример декларации адаптера для функции (класса) DemoFunction: 

package ru.grw.demo.avalanche;

import java.util.Hashtable;

import ru.funsys.avalanche.AvalancheRemote;

/**
 * Определение адаптера 
 * 
 * @author Валерий Лиховских
 *
 */
public interface DemoAdapter {

	public Hashtable<String, String> getInfo() throws AvalancheRemote;
	
}

Адаптеры используются классами приложений. Для подключения адаптера в приложении необходимо добавить в конфигурационный файл в секцию <application> секцию <adapter> (полностью секция <application> приведена в описании "Реализация приложения"). 

	<adapter class="ru.grw.demo.avalanche.DemoAdapter" name="info" uri="info-function" />

Где, атрибут name определяет имя адаптера, под которым экземпляр класса DemoAdapter будет известен классу приложения. Атрибут uri определяет имя локальной или удаленной функции. Для адресации локальной функции указывается значение атрибута name требуемой функции. Для адресации удаленной функции указывается пара значений через символ разделитель "/", первое значение определяет значение атрибута name используемого интерфейса, второе определяет значение атрибута name удаленной функции, опубликованной в удаленном коннекторе.

Пример адресации удаленной функции: 

	<adapter class="ru.grw.demo.avalanche.DemoAdapter" name="info" uri="rmi-interface/info" />

Реализация JSP-страницы (каскадный вызов)

В модификации кода страницы JSP так же нет какой либо необходимости. Для сохранения работоспособности ранее реализованной JSP страница, сделана ее другая копия, в которой необходимо изменить имя используемого экземпляра класса приложения (Application) c DemoApp на CascadeApp.

Повторные запросы этой страницы будут приводить к вызову функции getInfo() c другого сервера. 

<%@ page import="ru.grw.demo.avalanche.DemoApplication"%>
<%@ page language="java" contentType="text/html;charset=UTF-8"
    pageEncoding="UTF-8"%>
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
<html>
<head>
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
<title>Демонстрационное приложение - каскадный вызов</title>
</head>
<body>
<h1>Демонстрационное приложение - каскадный вызов</h1>
<jsp:useBean id='CascadeApp' scope='application' class='ru.grw.demo.avalanche.DemoApplication'/>

<% = CascadeApp.getInfo(true) %>

</body>
</html>

Экземпляр класса Application с именем CascadeApp использует адаптер, сконфигурированный на вызов локальной функции cascade-info, которая в свою очередь последовательно передает вызовы одной из трех вариантов вызовов функции info.
